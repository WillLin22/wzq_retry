# 仓库简介

Hi!作为助教，前两天看了自己当年写的五子棋史山后倍感无力，因此产生了“无AI，每次写两小时左右，看看能写成什么样子”的想法。联想到当年自己苦恼于完全不知道“什么样的程序是好程序”以及“如何将写好程序的原则（如做好隔离封装、模块化、代码复用等）落实到我的五子棋中”，于是将这些代码开源出来供大家学习。

总之，希望读者可以借此学习一下项目的结构设计和功能封装？当然这毕竟是我（一个人类）利用闲暇时间写的代码，难免有不完善的地方，希望大家能够取长补短，以及欢迎大家提issue和PR。


# 仓库框架
本仓库参照常规C项目目录结构：
- `build/`：存放构建生成的文件，不上传到版本库
- `src/`：存放源代码
- `include/`：存放头文件
- `tests/`：存放测试代码
- `docs/`：存放文档
- `Makefile`：构建脚本
- `README.md`：项目说明文件

# 构建与运行
1. 克隆仓库到本地：
    ```bash
    git clone 
    ```
2. 进入项目目录：
    ```bash
    cd wzq-retry
    ```
3. 编译源文件：
    ```bash
    make
    ```

4. 运行程序：
    ```bash
    make run
    ```
